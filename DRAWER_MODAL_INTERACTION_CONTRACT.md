## 1. Executive Decision
**Artefact status:** `DRAWER_MODAL_INTERACTION_CONTRACT_ACCEPTED_WITH_DOWNSTREAM_DEPENDENCIES`

**Interaction behaviour gate status:** `PARTIAL_NOT_PASSED`

**Codex readiness status:** `CODEX_HANDOFF_NOT_READY`

This artefact specifies drawer, modal, overlay, confirmation, wizard, upload and drawer-like side-panel interaction contracts for AlphaVest WealthOS. It converts the prior route, visual, state-screen and interaction-reality findings into behaviour contracts without implementing anything. It preserves the core finding from the interaction audit: component presence and visible drawer/modal surfaces are not lifecycle proof. The strongest current interaction remains document upload on route `028`; many other interactions are partial, deterministic, static or visual-state driven and therefore require downstream feedback, safety, API, schema and test proof before Codex can implement.

**Core decision:** MVP and flow-relevant MVP_SUPPORT overlays receive lifecycle contracts. P1 routes are deferred, reference-only routes are excluded from product interaction contracts, and hold routes remain blocked. No code, screen, state-screen, image, API, Prisma, test, Codex task or final handoff is produced.

## 6. Interaction Reality Vocabulary
| Label | Meaning | Codex Implication |
| --- | --- | --- |
| IMPLEMENTED_INTERACTION | Trigger, handler, visible result and at least basic success/error behaviour are code-present. | Still not Codex-ready when safety, audit, RBAC or evidence sufficiency is unproven. |
| PARTIALLY_IMPLEMENTED_INTERACTION | Some trigger/handler/result exists, often demo-only or incomplete. | Requires lifecycle, validation, permission, audit and test contracts. |
| DETERMINISTIC_VISUAL_STATE | Route/query/default state renders a surface without proving user-triggered lifecycle. | Must not be treated as implemented behaviour. |
| STATIC_SCREEN_STATE | JSX renders a panel/card/drawer-like area or disabled/status UI without state transition proof. | Needs behaviour spec before implementation. |
| COMPONENT_PRIMITIVE_ONLY | Reusable component exists but route-level behaviour is not proven. | Primitive must be bound to route contracts later. |
| VISUAL_STATE_SUPPORT_ONLY | Component can display a state but does not create or verify it. | Needs API/gate/source proof. |
| SPECIFIED_ONLY | Required by product/control logic but not proven in code. | Can become future task only after all contracts pass. |
| MISSING_BEHAVIOUR | Required behaviour is absent or unproven. | Route to missing behaviour register. |
| NEEDS_BEHAVIOUR_SPEC_BEFORE_CODEX | Not safe for implementation until lifecycle and safety expectations are explicit. | Codex blocked. |
