# Roadmap Scope Model v2

## Purpose

The roadmap route separates MVP, Phase 2 and future scope so the click-dummy does not imply future automation is already implemented.

## MVP

- Client dashboard and mobile upload.
- Action board and decision room.
- Evidence vault and audit mapping.
- Internal review, advisor approval and compliance release.
- Communication routing and service blueprint reference.

## Phase 2

- Deeper CRM and document-system integration.
- Expanded external specialist coordination.
- Configurable family-governance workflows.
- Operational reporting and SLA dashboards.

## Future

- Advanced AI assistance under human review.
- Multi-family operating model.
- Jurisdiction-aware automation.
- Partner ecosystem extensions.

## Blocked / Not MVP Ready

| Feature | Reason | Dependency |
|---|---|---|
| Autonomous advice | Advice-like output must remain human reviewed and compliance released. | Advisor and compliance gates |
| Direct client release from draft | Draft and analyst-reviewed content cannot become client visible. | No-unapproved-advice gate |
| External specialist self-serve access | Sensitive data sharing needs scoped permission and audit. | Second confirmation and access model |
| Unverified data automation | Evidence replaces assumption; low confidence must block or return. | Evidence validation flow |

## Dependency Flow

Source data -> evidence record -> analyst review -> advisor approval -> compliance release -> client visibility -> decision or communication log.
