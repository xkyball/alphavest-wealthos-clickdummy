Reader orientation

How to read this manual

AlphaVest WealthOS is a collaboration system for turning fragmented client, document, recommendation, governance, export, and operations data into controlled workflow state.

Process first

Each workflow begins with a role/state process graphic so the reader knows what is happening before reading UI detail.

Screenshot second

Annotated screenshots show where the process appears and connect numbered regions to procedure explanations.

Gate-aware

Advisor review, compliance release, permission scope, evidence lineage, and audit boundaries are explained as workflow controls.

Demo-aware

The manual distinguishes current demo surfaces from production persistence, certification, and full PDF/UA accessibility claims.

Codex Visual Implementation Standard

This V4 PDF is produced under the Codex Visual Implementation Standard: rendered pages, contact sheet, visual self-review, and explicit limitations are required before acceptance.

Process legend

The visual grammar used throughout the manual

The same symbols repeat across every workflow so process graphics and annotated screenshots reinforce each other instead of becoming separate artifacts.

Numbered markers

Numbers in the process graphic correspond to numbered regions in the annotated screenshot and nearby callout cards.

Solid and dashed paths

Solid lines indicate current documented flow. Dashed paths are reserved for future, fallback, or demo-boundary behavior.

Boundaries

Lock, visibility, audit, and evidence markers show why a step changes what another role can see or do next.
