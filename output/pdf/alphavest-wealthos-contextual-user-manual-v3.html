Manual frame

How to read this manual

Task-led, not path-led

The manual follows visible UI tasks and user outcomes. Route paths remain source trace material, not operating instructions.

Context beside action

Each flow explains the reason for the action, the data it depends on, the controls it activates, and the downstream workflow it prepares.

Evidence boundary

Where a feature is navigable but not fully production-persistent, the manual says so. Demo behavior is never inflated into a production claim.

Product purpose

  • AlphaVest WealthOS supports demo-data-first wealth operations for family-office and advisory workflows.
  • The product model connects digital intake, human review, evidence-backed decisions, role-scoped access, redaction, audit, and compliance-controlled client visibility.
  • The manual context layer teaches why actions exist, what they protect, and what later workflows depend on.

Safety principles

  • No unapproved advice reaches the client.
  • Advisor approval alone is not enough.
  • Compliance release controls client visibility.
  • Evidence and audit are required where the app claims important actions are complete.
  • Blocked states are protective controls, not generic errors.

Demo boundaries

  • Demo session is not production authentication.
  • Visible UI does not prove full persistence unless the source package states it.
  • File extraction, export package generation, share links, and broad governance transactions remain partial unless proven.
  • No real client data or final financial, legal, or tax advice is introduced.

Evidence model

Why blocked states and release gates are part of the workflow

Compliance release

Client visibility is controlled by compliance release, especially when content may influence financial, legal, or tax decisions.

Blocked states

A blocked state is a protective control. It tells the user what condition is missing instead of allowing an unsafe completion claim.
